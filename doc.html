پروژه دندون یار کوچولو

(Little Tooth Buddy)

مستند فنی، تشریح معماری سیستم و راهنمای کامل استفاده از سامانه

۱. مقدمه و اهداف پروژه

بهداشت دهان و دندان کودکان یکی از حیاتی‌ترین بخش‌های سلامت رشد آن‌هاست. با این وجود، تشویق کودکان به مسواک‌زدن و مراقبت مداوم روزانه معمولاً به یکی از چالش‌های والدین تبدیل می‌شود. پروژه «دندون یار کوچولو» با هدف حل این چالش از طریق گیمیفیکیشن (بازی‌وارسازی) و ایجاد یک محیط جذاب بصری برای کودکان طراحی و پیاده‌سازی شده است.

این پلتفرم به کودکان کمک می‌کند تا فرآیندهای روزانه مانند مسواک‌زدن، استفاده از دهان‌شویه و نخ دندان را در قالب بازی و دریافت امتیاز انجام دهند. از طرف دیگر، این سیستم ابزارهای نظارتی دقیقی را در اختیار والدین قرار می‌دهد تا بتوانند روند بهداشت دهان فرزندان خود را پیگیری، کنترل و با اهداف سفارشی هدایت کنند.

اهداف اصلی سیستم:

  • ایجاد عادت‌های مثبت بهداشت دهان و دندان در کودکان با استفاده از تکنیک‌های تعاملی و زمان‌سنج‌های جذاب.
  • فراهم آوردن پنل مدیریتی برای والدین جهت پایش دقیق عملکرد فرزندان و تعریف اهداف شخصی‌سازی شده.
  • ذخیره‌سازی و پایش تصویری دندان‌های کودک در یک بازه زمانی طولانی (گالری سیر تغییرات).
  • آموزش شیوه‌های صحیح مراقبت به وسیله ویدیوها و نکات آموزشی در درون اپلیکیشن.

۲. ساختار و معماری سیستم

پلتفرم دندون یار کوچولو به صورت یک ساختار کلاینت-سرور پیاده‌سازی شده است و شامل دو بخش اصلی فرانت‌اند (اپلیکیشن موبایل) و بک‌اند (وب‌سایت مدیریتی و وب سرویس کلاینت) می‌باشد.

۲.۱. بخش فرانت‌اند (Flutter App)

Flutter 3.9+ Dart Dio HTTP Video Player (FVP) Image Picker YekanBakh Font

بخش کلاینت سیستم یک اپلیکیشن موبایل چندسکویی (Cross-Platform) است که با استفاده از فریم‌ورک محبوب فلاتر (Flutter) توسعه یافته است. این کلاینت وظیفه ارائه تجربه کاربری پویا، انیمیشن‌های روان و هدایت کودک در فرآیند بهداشت را بر عهده دارد. ارتباط فرانت‌اند با سرور از طریق کتابخانه قدرتمند Dio و به صورت RESTful انجام می‌پذیرد.

۲.۲. بخش بک‌اند (ASP.NET Core Web API)

.NET 8.0/9.0 ASP.NET Core Web API Entity Framework Core SQLite Database Identity Core JWT Bearer Auth Autofac Serilog

بخش سرور سیستم بر پایه پلتفرم پرقدرت ASP.NET Core Web API توسعه داده شده است. معماری این بخش بر اساس اصول معماری تمیز (Clean Architecture) پایه‌گذاری شده که لایه‌های مختلف را به شرح زیر تفکیک می‌کند:

  • لایه دامنه (Domain): شامل قوانین پایه کسب‌وکار، انام‌ها (Enums) و متدهای الحاقی اصلی است.
  • لایه برنامه (Application): شامل موجودیت‌های دیتابیس (Entities)، کلاس‌های نگاشت داده (DTOs/Models)، اینترفیس‌های سرویس‌ها و فایل‌های مربوط به مهاجرت‌های دیتابیس (Migrations) است.
  • لایه زیرساخت (Infrastructure): شامل پیاده‌سازی سرویس‌ها (نظیر احراز هویت JWT، مدیریت مستأجرین و منطق کسب‌وکار دندون یار) است.
  • لایه API (HealthcareApi): لایه ورودی و کنترلرها که درخواست‌های HTTP را دریافت کرده، اعتبارسنجی می‌کند و خروجی مناسب را بازمی‌گرداند.

۳. امکانات و سناریوهای کاربری فرانت‌اند

اپلیکیشن دندون یار کوچولو شامل صفحات و سناریوهای کاربری مختلفی است که تعامل بین والد، کودک و سرور را برقرار می‌سازد:

۱. ورود و عضویت والدین (Auth Screen)

والدین به عنوان کاربر اصلی ثبت‌نام می‌کنند. پس از وارد کردن شماره تلفن و رمز عبور، یک توکن JWT دریافت شده و کلاینت در تمامی درخواست‌های بعدی از آن استفاده می‌کند.

۲. داشبورد و مدیریت فرزندان (Dashboard & Children Management)

پس از ورود، والدین می‌توانند پروفایل کودکان خود را (نام، سن، آواتار) مدیریت کنند. کودک فعال در اپلیکیشن انتخاب می‌شود و داشبورد تعاملی مخصوص او لود می‌گردد.

۳. زمان‌سنج و راهنمای تعاملی مسواک‌زدن (Interactive Brushing)

هسته اصلی بخش کودکان، زمان‌سنج مسواک‌زدن تعاملی است. این بخش کودک را گام به گام (مثلاً مسواک‌زدن جلو، چپ، راست و دندان‌های بالا و پایین) به همراه تایمر راهنمایی می‌کند تا فرآیند مسواک‌زدن را به شکل صحیح و در مدت زمان استاندارد به پایان برساند. در انتهای فعالیت، کودک ستاره دریافت می‌کند.

۴. بخش نخ دندان و دهان‌شویه (Flossing & Mouthwash)

کودکان در این بخش یاد می‌گیرند که استفاده از نخ دندان و دهان‌شویه را پس از مسواک‌زدن به عنوان مکمل‌های بهداشتی ثبت کنند و ستاره‌های بیشتری به دست آورند.

۵. آلبوم تصاویر دندان‌ها (Teeth Gallery)

کودک یا والد با دوربین موبایل از دندان‌های کودک عکس می‌گیرند. این تصاویر با عنوان مشخص آپلود شده و یک گالری تاریخی برای دندانپزشک یا والد تشکیل می‌دهند تا روند بهبود دندان‌ها پایش شود.

۶. مدال‌ها، اهداف و سیستم پاداش‌ها (Badges & Goals System)

با تکرار منظم کارها، کودک نشان‌ها و مدال‌های مختلف را باز می‌کند. همچنین والدین اهدافی (مثلاً ۱۰ بار مسواک‌زدن) تعریف می‌کنند که با رسیدن به آن‌ها، کودک امتیاز پاداش دریافت می‌کند. در پنل پاداش‌ها، کودک می‌تواند ستاره‌های خود را خرج کند تا پاداش‌های واقعی که والدین تعریف کرده‌اند (مانند رفتن به پارک یا خرید اسباب‌بازی) را به عنوان درخواست ثبت کند و والدین آن را تایید نمایند.

۴. ساختار پایگاه داده و بک‌اند

سرور پروژه از دیتابیس **SQLite** استفاده می‌کند که به راحتی پرتابل و قابل جابجایی است. نقشه‌ها و روابط جداول به طور کامل توسط لایه Entity Framework Core مدیریت می‌شوند.

۴.۱. موجودیت‌های کلیدی پایگاه داده (Entities)

نام جدول/موجودیت کلیدهای اصلی/خارجی شرح و کاربرد
Child Id (Guid), ParentId نگهداری اطلاعات فرزند شامل نام، سن، لینک آواتار و تعداد ستاره‌های فعلی. هر کودک به یک کاربر والد متصل است.
ActivityLog Id (Guid), ChildId ثبت گزارش فعالیت‌های بهداشتی کودک شامل نوع فعالیت (مسواک، نخ دندان، دهان‌شویه)، مدت زمان فعالیت به ثانیه، تاریخ انجام و ستاره‌های کسب شده.
Goal Id (Guid), ChildId اهداف تعریف‌شده برای هر فرزند (مثلاً مسواک زدن‌های متوالی) به همراه وضعیت پیشرفت و تعداد ستاره‌های جایزه.
Reward Id (Guid), ChildId پاداش‌های قابل دریافت توسط کودک که توسط والدین تعریف می‌شود و نیاز به تعداد مشخصی ستاره دارد.
RewardClaim Id (Guid), ChildId, RewardId درخواست‌های ثبت‌شده توسط کودک برای قفل‌گشایی یک جایزه. دارای وضعیت تعلیق (Pending) یا تایید شده (Approved).
Badge Id (Guid), ChildId نشان‌ها یا دستاوردهایی که کودک به واسطه استمرار در بهداشت به صورت خودکار باز کرده است.
TeethPhoto Id (Guid), ChildId عکس‌های آپلود شده از دندان‌های کودک برای مقایسه روند رشد و بهداشت دندان‌ها.
AppConfig Id (Int) تنظیمات عمومی کل سیستم شامل لینک ویدیو آغازین، پیغام روز (MOTD)، شماره تماس پشتیبانی و نسخه نرم‌افزار.

۵. راهنمای گام‌به‌گام کارکرد سامانه و سناریوهای عملیاتی

سیستم «دندون یار کوچولو» به گونه‌ای طراحی شده است که فرآیندهای بهداشتی دهان و دندان را با بازی‌وارسازی (گیمیفیکیشن) برای کودک سرگرم‌کننده و برای والدین کاملاً قابل پایش کند. در این بخش، نحوه کارکرد عملیاتی سیستم و سناریوهای اجرایی در اپلیکیشن موبایل و داشبوردهای سمت سرور به طور دقیق تشریح می‌گردد.

۵.۱. نحوه کارکرد و جریان داده کلی سامانه

ارتباط فرانت‌اند و بک‌اند بر بستر توکن‌های امنیتی JWT تعریف شده است. والد ابتدا وارد اپلیکیشن شده و پروفایل کودک را می‌سازد. پس از آن، تمام تعاملات کودک با برنامه (از جمله زمان‌سنج مسواک، نخ دندان، دهان‌شویه و گالری عکس) تحت شناسه اختصاصی هر کودک به سرور ارسال و در دیتابیس ثبت می‌شود. سرور با تحلیل مداوم این لاگ‌ها، آمار روزهای متوالی (Streaks)، اهداف تحقق‌یافته و نشان‌های تشویقی باز شده را به صورت بلادرنگ به کلاینت بازمی‌گرداند.

۵.۲. سناریوها و قابلیت‌های اپلیکیشن موبایل (سمت کلاینت)

۱. سناریو ثبت‌نام، ورود و انتخاب کودک (پروفایل والدین)

والدین در اولین ورود به برنامه، با شماره موبایل و کلمه عبور ثبت‌نام می‌کنند. پس از ورود، اپلیکیشن به صورت خودکار توکن امنیتی را از سرور دریافت و ذخیره می‌کند. در گام بعدی، والد می‌تواند یک یا چند فرزند با نام، سن و آواتار دلخواه ایجاد کند. با انتخاب هر کودک، محیط اپلیکیشن به حالت کودک تغییر وضعیت داده و فعالیت‌های بهداشتی او لود می‌شود.

۲. سناریو مسواک‌زدن تعاملی و زمان‌سنج هوشمند (Interactive Brushing)

هنگامی که کودک دکمه «مسواک‌زدن» را لمس می‌کند، با یک راهنمای انیمیشنی مواجه می‌شود. این صفحه طبق زمان استاندارد پزشکی (۲ دقیقه)، دهان را به چند بخش مجزا (دندان‌های جلو، چپ، راست، بالا، پایین و زبان) تقسیم می‌کند. راهنمای تصویری به کودک نشان می‌دهد در هر ثانیه کدام بخش را مسواک بزند. پس از پایان زمان‌سنج و ثبت موفق لاگ، سرور امتیازهای ستاره را به حساب کودک واریز می‌کند.

۳. سناریو ثبت نخ دندان و دهان‌شویه (Complementary Care)

در این بخش، کودک با فشردن دکمه دهان‌شویه یا نخ دندان، وارد فرآیند شبیه‌ساز یا تایمر مخصوص هرکدام می‌شود. این کار با هدف ترغیب کودک به اتمام چرخه کامل بهداشت دهان انجام می‌شود. انجام این وظایف امتیازات تکمیلی (ستاره‌های بیشتر) به همراه دارد که در سرور ثبت می‌گردد.

۴. سناریو ثبت و ارسال عکس دندان‌ها (Teeth Timeline Gallery)

کودک یا والد می‌توانند با دوربین گوشی از وضعیت دندان‌ها عکس بگیرند و با وارد کردن عنوان (مثلاً «قبل از عصب‌کشی» یا «مسواک زدن هفته اول») آن را به سرور بفرستند. تصاویر آپلود شده به صورت خط زمانی (Timeline) ذخیره شده و به والدین این امکان را می‌دهد تا تغییرات فیزیکی دندان‌ها را در درازمدت پایش کنند.

۵. سناریو خرج ستاره‌ها و تقاضای پاداش (Claiming Rewards)

کودک با مراجعه به ویترین جوایز، لیست پاداش‌هایی را که والدین برای او تعریف کرده‌اند (مثلاً «خرید اسباب‌بازی»، «یک ساعت بازی با کنسول») مشاهده می‌کند. هر جایزه نیاز به تعداد مشخصی ستاره دارد. در صورت داشتن ستاره کافی، کودک درخواست اهدای جایزه را ثبت می‌کند. در این مرحله، ستاره‌ها کسر شده و وضعیت درخواست در سرور به «در انتظار تایید والد» تغییر می‌یابد.

۵.۳. سناریوها و داشبوردهای سمت سرور (سمت والد و ادمین)

۱. داشبورد و پنل نظارتی والدین (Parent Control Panel)

  • پایش آماری کودکان: والد در پنل اختصاصی خود به نمودارهای فعالیت کودک دسترسی دارد. او می‌تواند متوجه شود فرزندش در چه روزهایی مسواک زده، آیا مسواک صبح یا شب را فراموش کرده و زنجیره استمرار او چند روزه بوده است.
  • تعریف اهداف سفارشی (Custom Goals): والدین می‌توانند اهداف ویژه‌ای تعریف کنند. به عنوان مثال: هدف «۱۰ بار مسواک زدن متوالی در ماه جاری» با جایزه «۲۰ ستاره طلایی». سیستم به صورت خودکار پیشرفت کودک را پایش و در صورت تکمیل هدف، ستاره‌ها را واریز می‌کند.
  • مدیریت جوایز واقعی: والدین پاداش‌های فیزیکی دنیای واقعی را متناسب با علایق کودک در پنل ثبت می‌کنند و هزینه ستاره‌ای آن را تعیین می‌نمایند.
  • تایید فیزیکی درخواست جوایز: پس از تقاضای جایزه توسط کودک، والد اعلانی دریافت می‌کند. والد پس از تحویل دادن اسباب‌بازی یا پاداش به کودک در دنیای واقعی، دکمه «تایید و پرداخت» را در پنل کلیک می‌کند تا فرآیند نهایی شود.

۲. پنل مدیریت سراسری سیستم (Global Admin Console)

  • مدیریت محتوای آموزشی (ویدیو و نکات علمی): مدیر ارشد سیستم می‌تواند ویدیوهای آموزشی جدید با فرمت MP4 بارگذاری کند. این ویدیوها در مسیر رسانه‌ای سرور قرار گرفته و در بخش ویدیوهای برنامه برای کودکان نمایش داده می‌شوند. همچنین، نوشتن نکات کوتاه علمی (Tips) جهت نمایش در صفحه اصلی کودکان از این پنل انجام می‌شود.
  • پیکربندی تنظیمات سیستم (App Config): تنظیم پیام آغازین برنامه (Splash Message)، بروزرسانی لینک دانلود نسخه‌های جدید اپلیکیشن، شماره پشتیبانی و غیرفعال‌سازی موقت بخش‌ها در مواقع نگهداری سرور.

۶. راهنمای کاربردی و آموزشی استفاده از سامانه (راهنمای اپراتوری)

این بخش به عنوان یک راهنمای عملیاتی و دفترچه راهنمای کاربر (User Guide) تدوین شده است تا افرادی که وظیفه کار با سیستم یا آموزش آن به کاربران نهایی را دارند، بتوانند به راحتی از تمام پتانسیل‌های نرم‌افزار استفاده نمایند.

۶.۱. راهنمای گام‌به‌گام کودکان در اپلیکیشن موبایل

بخش کودک با گرافیک فانتزی و ساده طراحی شده است تا تعامل با آن برای سنین پایین لذت‌بخش باشد. مراحل کار به شرح زیر است:

۶.۱.۱. مسواک‌زدن تعاملی و شبیه‌ساز مسواک (زمان‌سنج مسواک)

این ماژول از ۱۲ مرحله پیاپی تشکیل شده است که کودک را از انتخاب مسواک تا آب‌کشی نهایی راهنمایی می‌کند:

  1. انتخاب مسواک (Choose Brush): کودک ابتدا مسواک فانتزی دلخواه خود را از میان مسواک‌های فانتزی نمایش داده شده انتخاب می‌کند.
  2. انتخاب زمان (Choose Time): مدت زمان مسواک زدن (مثلاً حالت سریع ۱ دقیقه‌ای یا حالت استاندارد ۲ دقیقه‌ای) تعیین می‌شود.
  3. شستشوی دندان‌های جلو بالا (Front Upper): کودک باید با مسواک روی دندان‌های جلویی و بالایی در صفحه حرکت دایره‌ای (Circular gestures) انجام دهد. انجام این کار باعث تولید کف مسواک (Foam bubbles) روی دندان‌ها در صفحه می‌شود.
  4. شستشوی دندان‌های جلو پایین (Front Lower): کودک باید حرکت عمودی بالا به پایین (Vertical strokes) روی دندان‌های جلویی پایین انجام دهد.
  5. شستشوی بخش‌های داخلی دندان‌های عقب بالا (Back Upper Inner): راهنمایی کودک برای حرکت مسواک روی سطوح داخلی دندان‌های پشتی.
  6. شستشوی بخش‌های خارجی دندان‌های عقب بالا (Back Upper Outer): مسواک زدن بخش‌های بیرونی دندان‌های پشتی.
  7. شستشوی سطح جونده دندان‌های عقب بالا (Back Upper Chewing): کودک باید حرکت افقی رفت و برگشتی (Horizontal Chewing strokes) روی سطح جونده دندان‌های عقب انجام دهد تا جرم‌ها پاک شوند.
  8. شستشوی دندان‌های عقب پایین (Back Lower): پاکسازی دندان‌های پشتی پایینی با تلفیقی از حرکات افقی و دایره‌ای.
  9. مسواک‌زدن زبان (Brush Tongue): کودک باید با حرکت جاروبی رو به جلو (Downward swipes) روی زبان موجود در شبیه‌ساز بکشد تا جرم‌های زبان پاک شوند.
  10. ادامه تا پایان زمان استاندارد (Continue 2 Minutes): در صورتی که کار زودتر تمام شده باشد، تایمر تا پایان زمان استاندارد به کودک انیمیشن‌های تشویقی نشان می‌دهد.
  11. تُف کردن کف‌ها (Spit Out): پس از اتمام تایمر، از کودک خواسته می‌شود روی دکمه ضربه بزند تا کف جمع شده در دهان خود را در شبیه‌ساز خالی کند (تف کند).
  12. شستشوی دهان و اتمام (Clean Mouth Done): آب‌کشی نهایی انجام شده و جشن پیروزی با افکت رقص ستاره‌ها (Confetti) لود می‌شود. کودک دکمه «ثبت فعالیت» را می‌زند تا اطلاعات بلافاصله در سرور ذخیره و ستاره‌ها واریز شوند.

۶.۱.۲. شبیه‌ساز و بازی تعاملی نخ دندان (Flossing Game)

استفاده از نخ دندان به صورت یک بازی فکری و سرگرم‌کننده طراحی شده است:

  1. از منوی اصلی دکمه مربوط به «نخ دندان» را لمس کنید.
  2. در صفحه بازی، مدل دندان‌ها نمایش داده می‌شود که در فواصل بین آن‌ها، ذرات غذا (سبزی، گوشت، پنیر) و میکروب‌های مزاحم گیر افتاده‌اند.
  3. کودک باید با انگشت خود نخ دندان را لمس کرده و با کشیدن انگشت (Drag Gesture)، نخ را دقیقاً به درون فاصله‌های دندانی هدایت کند.
  4. با عبور موفق نخ دندان از هر فاصله، ذرات غذا متلاشی شده، انیمیشن ذرات و جرقه فعال می‌شود و میکروب‌ها نابود می‌گردند.
  5. با پاکسازی هر فاصله، نوار پیشرفت تمیزی دندان‌ها (Cleanliness progress) افزایش می‌یابد. پس از پاکسازی کامل تمام فواصل (۱۰۰٪)، سیستم لاگ فعالیت flossing را در سرور ثبت کرده و امتیاز ستاره‌ها را به کودک اعطا می‌کند.

۶.۱.۳. شبیه‌ساز و بازی تعاملی دهان‌شویه (Mouthwash Gargle)

این بخش به کودک یاد می‌دهد چگونه دهان خود را پس از پاکسازی اولیه ضدعفونی کند:

  1. گزینه «دهان‌شویه» را در داشبورد انتخاب کنید.
  2. یک بطری دهان‌شویه فانتزی و یک لیوان روی صفحه ظاهر می‌شود. کودک باید دکمه فشاری بطری را با انگشت خود نگه دارد (Press and Hold).
  3. با نگه داشتن دکمه، مایع دهان‌شویه با افکت فیزیک آب به آرامی لیوان را پر می‌کند. اگر کودک انگشت خود را بردارد، مایع داخل لیوان دوباره به آرامی تخلیه می‌شود (Drain).
  4. کودک باید دکمه را آن‌قدر نگه دارد تا سطح مایع لیوان به ۱۰۰٪ (خط پر شدن) برسد.
  5. سپس انیمیشن قرقره کردن ۳۰ ثانیه‌ای به صورت خودکار آغاز می‌شود. پس از پایان ضدعفونی، با نمایش یک پیام شاداب‌کننده («به‌به! عجب بوی خوبی! دهانت بوی طراوت نعناع میده! 🍃») و افکت‌های ستاره، فعالیت mouthwash ثبت شده و امتیاز آن واریز می‌گردد.

۶.۱.۴. آلبوم عکس و سیر پیشرفت تغییرات دندان (Teeth Photo Timeline)

پایش بصری دندان‌ها به این صورت انجام می‌شود:

  1. وارد بخش «آلبوم عکس دندان» شوید.
  2. روی دکمه دوربین یا آپلود عکس کلیک کنید.
  3. عنوان عکس را وارد کنید (مثلاً: افتادن اولین دندان شیری، هفته اول ارتودنسی).
  4. با دوربین گوشی عکس واضحی از دندان کودک بگیرید. پس از تایید، عکس به سرور آپلود شده و به صورت خط زمان مرتب می‌شود.
  5. والدین و دندانپزشکان می‌توانند با مقایسه عکس‌های تاریخی، روند رشد یا ترمیم دندان‌ها را به دقت بررسی کنند.

۶.۲. راهنمای کاربردی و تنظیمات مخصوص والدین در اپلیکیشن موبایل

والدین از طریق بخش تنظیمات اختصاصی می‌توانند نظارت کاملی بر روند بهداشت داشته باشند:

۶.۲.۱. تنظیم زمان یادآوری مسواک صبح و شب (Reminder Settings)

برای تنظیم هشدارها جهت یادآوری به موقع مسواک به کودک:

  1. وارد صفحه تنظیمات شوید و روی گزینه «یادآوری مسواک صبح و شب» کلیک کنید.
  2. در پنجره باز شده، تیک فعال‌سازی اعلان‌ها را فعال نمایید.
  3. ساعت یادآوری صبح (مثلا ۰۸:۰۰) و ساعت یادآوری شب (مثلا ۲۱:۰۰) را به صورت دقیق تنظیم کنید.
  4. دکمه ثبت را بزنید. این تنظیمات در سرور ذخیره شده و کلاینت در ساعت‌های مشخص شده به صورت نوتیفیکیشن هشدار می‌دهد.

۶.۲.۲. مدیریت اهداف سفارشی (Goals Manager)

برای تشویق استمرار فعالیت کودک:

  1. به بخش «مدیریت اهداف» در پنل والدین بروید.
  2. گزینه «ایجاد هدف جدید» را بزنید.
  3. عنوان (مثلاً: مسواک طلایی دو هفته‌ای)، شرح هدف و تعداد دفعات انجام (مثلاً ۱۴ بار) را مشخص کنید.
  4. تعداد ستاره جایزه (مثلاً ۳۰ ستاره) را ثبت کنید. با استمرار فعالیت‌های کودک، نوار پیشرفت در داشبورد لود شده و پس از تکمیل، پاداش به کودک اعطا می‌شود.

۶.۲.۳. تعریف پاداش‌های فیزیکی واقعی و تایید درخواست‌ها (Rewards Management)

والدین می‌توانند ستاره‌های مجازی کودک را به پاداش‌های واقعی در خانه پیوند بزنند:

  1. تعریف پاداش: در پنل والد، به بخش جوایز رفته و دکمه ایجاد پاداش را بزنید. عنوان پاداش (مانند خرید کتاب داستان، رفتن به پارک تفریحی یا نیم ساعت بازی کامپیوتری) و هزینه ستاره‌ای آن (مثلاً ۵۰ ستاره) را بنویسید و ثبت کنید.
  2. تایید درخواست: وقتی کودک در اپلیکیشن جایزه‌ای را با ستاره‌هایش خریداری می‌کند، وضعیت درخواست در سرور به حالت معلق (Pending) درمی‌آید. والد در پنل خود لیست درخواست‌های در انتظار را بررسی می‌کند. پس از تحویل پاداش واقعی به کودک، روی گزینه «تایید و اهدای پاداش» کلیک می‌کند تا امتیازات به طور قطعی کسر شده و درخواست آرشیو شود.

۶.۲.۴. مدیریت و ویرایش پروفایل کودک (Profile Update)

برای تغییر نام، سن یا آواتار کودک فعال در برنامه:

  1. در بخش تنظیمات، روی دکمه «تغییر اطلاعات پروفایل کودک» کلیک کنید.
  2. نام جدید، سن به‌روزشده یا آدرس آواتار جدید را ثبت نمایید.
  3. پس از ذخیره، تمامی آمار و ستاره‌های فرزند در ساختار جدید لود خواهد شد.

۶.۳. راهنمای کاربردی و مدیریت سامانه مخصوص مدیران سیستم (Admin Panel)

مدیر سیستم کنترل کاملی بر روی منابع رسانه‌ای و تنظیمات کلی اپلیکیشن کلاینت‌ها دارد:

۶.۳.۱. بارگذاری ویدیوهای آموزشی جدید

  1. به بخش مدیریت ویدیوها در پنل بک‌اند مراجعه کنید.
  2. عنوان ویدیو، توضیحات کوتاه آموزشی و مدت زمان ویدیو را وارد کنید.
  3. فایل ویدیو (با فرمت MP4) را انتخاب کنید تا به طور خودکار در مسیر سرور (wwwroot/uploads/videos) بارگذاری شده و آدرس مستقیم دریافت کند.
  4. دکمه ثبت را بزنید. ویدیو به صورت آنی در تمام اپلیکیشن‌های موبایل در بخش ویدیوهای آموزشی سلامت دهان ظاهر می‌شود.

۶.۳.۲. مدیریت و ثبت نکات آموزشی روزانه (Tips)

  1. به بخش مدیریت نکات بروید. عنوان نکته، متن کوتاه بهداشتی و دسته‌بندی آن (مثل مسواک زدن، نخ دندان، تغذیه) را بنویسید.
  2. در صورت تمایل تصویر مرتبط را آپلود کنید.
  3. با ثبت نهایی، این نکات به عنوان بنر آموزشی یا پیام روز در بالای داشبورد کلاینت‌ها برای کودکان نمایش داده می‌شود.

۶.۳.۳. تنظیمات سراسری سیستم (Global Configuration)

  1. برای به‌روزرسانی پیغام خوش‌آمدگویی سراسری (MOTD)، تغییر حداقل نسخه مجاز اپلیکیشن جهت اجبار به آپدیت، یا تغییر شماره پشتیبانی، به بخش تنظیمات سیستم در پنل بک‌اند بروید.
  2. مقادیر جدید را وارد و دکمه بروزرسانی را بزنید. کلاینت‌ها در اولین ارتباط با فراخوانی اندپوینت /config، تنظیمات جدید را اعمال خواهند کرد.

۷.۳. نیازمندی‌های سیستم

  • برای فرانت‌اند: Flutter SDK v3.9+ به همراه Android Studio یا Xcode جهت شبیه‌سازها.
  • برای بک‌اند: .NET Core SDK v8.0 یا v9.0 نصب شده روی سرور یا سیستم محلی.

۷.۴. اجرای پروژه بک‌اند (.NET Web API)

۱. وارد پوشه بک‌اند پروژه شوید: 

cd /mnt/sda2/porjects/dotnet/HealthcareApi/HealthcareApi/

۲. پکیج‌های پروژه را بازیابی کنید: 

dotnet restore

۳. دیتابیس و مهاجرت‌ها را اعمال کنید (پروژه به صورت خودکار در اولین اجرا دیتابیس SQLite را می‌سازد و داده‌های اولیه را تزریق می‌کند. اما در صورت نیاز دستی): 

dotnet ef database update --project HealthcareApi.Application/ --startup-project HealthcareApi/

۴. پروژه را در حالت توسعه یا تولید اجرا کنید: 

dotnet run --project HealthcareApi/

بک‌اند معمولاً روی پورت http://localhost:5158 یا پورت مشخص‌شده در appsettings.json لود می‌شود. برای مشاهده مستندات تعاملی Swagger به آدرس http://localhost:5158/swagger مراجعه کنید.

۱. ابتدا وارد پوشه پروژه فلاتر شوید: 

cd /mnt/sda2/porjects/flutter/healthcare

۲. پکیج‌های وابستگی کلاینت را دریافت کنید: 

flutter pub get

۳. مطمئن شوید که آدرس IP یا دامنه بک‌اند در فایل lib/api/healthcare_api.dart به درستی تنظیم شده باشد: 

static final ApiClient _defaultClient = ApiClient(baseUrl: 'https://h.ghahremansalamat.ir');

۴. برنامه را روی شبیه‌ساز یا دستگاه متصل اجرا کنید: 

flutter run

۸. جمع‌بندی نهایی و وضعیت تحویل

پروژه دندون یار کوچولو (ToothBuddy) با معماری استاندارد لایه‌ای در بک‌اند و کلاینت فلاتر بسیار روان، کلیه امکانات مورد نیاز کارفرما را پوشش داده است.

بخش بک‌اند به دلیل استفاده از معماری تمیز و تزریق وابستگی‌های هوشمند (Autofac)، از قابلیت نگهداری (Maintainability) و توسعه‌پذیری بالایی برخوردار است. دیتابیس پروژه SQLite انتخاب شده که فرآیند پشتیبان‌گیری و انتقال را به یک کپی ساده فایل محدود می‌کند. بخش موبایل فلاتر نیز با استایل‌دهی فونت وزیر و زمان‌سنج تعاملی آماده بهره‌برداری کامل است.

این مستند به عنوان راهنمای نهایی تحویل سیستم به کارفرمای محترم تقدیم می‌گردد. تمام سورس‌کدها به همراه دیتابیس نمونه آماده دپلویمنت نهایی بر روی سرورهای ابری هستند.
